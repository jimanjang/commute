# 🚀 [v2.0] 전사 근태 어드민 시스템 대규모 통합 및 최종 안정화 배포 노트 (Major Release)

본 릴리즈 버전 **[v2.0]**은 실시간 전사 근태 관리 시스템의 모든 기능적 고도화를 마치고, **운영 환경에서의 완벽한 신뢰성과 데이터 정밀성을 보장하는 최종 안정화 이정표(Milestone)**입니다. 

특히 오늘 발생했던 타임존 기반 알림 루프 이슈와 BigQuery 데이터 유실 사고를 전면 극복하고, 서비스 아키텍처의 내결함성(Fault Tolerance)을 한 차원 더 끌어올려 공식적인 **메이저 2.0 버전**으로 배포합니다.

## 🌟 핵심 안정화 및 복구 성과 (v2.0 Key Fixes)

### **1. 타임존 강인형(Timezone-Robust) 실시간 중복 발송 제어 엔진 탑재**
* **기존 문제점**: 기존의 `CONVERT_TZ` 기반 중복 체크 로직은 MySQL 세션 타임존 설정에 따라 **오후 3:00 KST**를 기점으로 다음 날짜로 잘못 연산되어 실시간 알림 중복 감지에서 제외되는 치명적인 버그가 있었습니다. 이로 인해 특정 사용자(Kaya, Rhea 등)에게 30초마다 슬랙 알림이 무한 스팸 발송되는 현상이 발생했습니다.
* **v2.0 해결책**: 의존성이 복잡한 타임존 연산 대신, 데이터베이스 타임존 세션에 영향을 받지 않고 인덱스를 직접 타는 **날짜 경계값 절대 비교 방식(`created_at >= 'YYYY-MM-DD 00:00:00'`)**으로 쿼리를 개선하였습니다.
* **적용 파일**: 
  - `src/app/api/admin/bot/triggers/run/route.ts`
  - `src/app/api/admin/bot/triggers/gaps/route.ts`
* **결과**: 특정 시간대(오후 3시 이후)에도 완벽한 중복 방지가 이루어져 알림 노이즈가 0%로 수렴하게 되었습니다.

### **2. BigQuery 데이터 유실 복구 및 대시보드 완전 정상화**
* **기존 문제점**: BigQuery Dataset 내의 핵심 테이블인 `secom-data.secom.workhistory`가 유실/삭제되어, 대시보드의 실시간 근태 현황 API(`/api/users`)가 500 에러를 뿜으며 화면상의 모든 구성원 수치(출근 대상, 출근 완료, 휴가 등)가 `0명`으로 마비되는 상태였습니다.
* **v2.0 해결책**: 로컬 MySQL의 백업 마스터 테이블(`t_secom_workhistory`)로부터 과거 근태 데이터 **38,036건**을 정밀 추출한 뒤, `pandas_gbq` 파이프라인을 통하여 BigQuery에 완벽하게 복원/재적재하였습니다.
* **결과**: 대시보드의 실시간 근태 통계(102명 출근 대상자, 100명 출근 완료 등) 및 월간 캘린더 근태 현황판이 100% 실시간으로 아름답게 시각화 복구되었습니다.
